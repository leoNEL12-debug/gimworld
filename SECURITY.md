## Reporting a Vulnerability

If you discover a security vulnerability in the gimworld site or repository, please report it responsibly.

### How to Report

**Do NOT open a public issue.** Instead, contact us:

- 📧 **Email:** Use our [contact form](https://forms.office.com/r/kFyH7TXtQJ)
- 📝 **Subject line:** "Security Vulnerability Report"

### What to Include

Please provide:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

### Response Time

- We aim to acknowledge reports within **48 hours**
- We'll provide updates on the fix within **7 days**
- Once fixed, we'll credit you (unless you prefer to remain anonymous)

## Security Best Practices

For gimworld community members:

- Never share your personal information publicly
- Use the support contact form for sensitive issues
- Report suspicious activity or spam immediately
- Keep your GitHub/Padlet accounts secure with strong passwords

## Scope

This policy covers:
- The gimworld GitHub Pages site
- This repository and its code
- Links and integrations (Padlet, contact form)
